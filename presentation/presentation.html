The Problem

  • Unplanned failures cost $50,000+ per incident — emergency repair, production halt, safety risk
  • Scheduled maintenance wastes resources — healthy engines pulled offline at fixed intervals
  • No visibility into actual equipment health — decisions based on calendar, not condition
The question:
Can we predict when an engine will fail
using sensor data alone?
Cost of Unplanned Failure
$50,000
Emergency repair + production downtime
Cost of Planned Maintenance
$5,000
Scheduled replacement during downtime
Cost Asymmetry
10 : 1
Missing a failure vs. an extra inspection

NASA CMAPSS Dataset

  • 100 turbofan engines run from healthy state to failure
  • 21 sensor readings per operating cycle (temperatures, pressures, speeds)
  • 20,631 time steps in training data
  • Engine lifespans: 128 to 362 cycles
  • Single operating condition (FD001 subset)
Source
NASA Prognostics Center
Most cited benchmark in predictive maintenance literature
Target Variable
Remaining Useful Life
Cycles until engine failure — capped at 130
Pipeline: Raw Sensors → Feature Engineering → Normalization → Train/Val Split by Engine → Model Training → Maintenance Decision

Recommendations for Plant Management

  • Deploy the predictive model for real-time fleet monitoring — RF for continuous RUL, KNN for binary alerts
  • Set alarm threshold at 10 cycles — optimal under 10:1 cost asymmetry
  • Prioritize sensors 4, 7, 11 — consensus top features across all importance methods
  • Expected ROI: 72% cost reduction — $361,500/year savings for 100-engine fleet
Limitations & Next Steps
  • Simulated data — validate on real operational data
  • Single operating condition — extend to FD002-FD004
  • Classical ML only — LSTM comparison as future work
  • Build real-time dashboard (Streamlit / FastAPI)
